## 定位

| | 他们 | 我们 |

---

## 三句话讲清楚

1. 扫描——34 条内置规则 + Bandit 100+ 底座，已知漏洞全覆盖
2. 审核——DeepSeek 本地 LLM 二审，40% 误报压到接近 0
3. 发现——LLM 读代码逻辑，找出「没有规则能描述但确实危险」的隐患【黑科技】

---

## 黑科技详解（LLM 无规则发现）

不是噱头，是具体流程：

```
scan(34规则) → ML filter → DS review → **LLM heuristic pass** → fix
                                              ↑
                                    prompt: "这段代码除了上述问题，
                                    还有哪些安全风险？请逐行审查。"
                                              ↓
                                    [Labs] 发现标记
                                    - 潜在竞态条件 (L42-L48)
                                    - 异常吞没风险 (L55)
                                    - 时间戳依赖 (L71)
```

**和规则扫描的区别：**
- 规则扫描：找「已知模式」→ eval(), pickle.loads(), shell=True
- LLM 启发：找「逻辑隐患」→ 竞态、时序、异常处理、权限检查缺失

**为什么不污染主结果：**
- LLM 发现单独放在 `[Labs]` 区
- 不做去重/不调整置信度
- 用户自主判断
